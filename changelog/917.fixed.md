Fix activation checkpointing being a no-op during finetuning of TabPFN v3 (and v2.5/v2.6), which caused CUDA OOM on long-context training. The legacy `model.recompute_layer` attribute only applied to the v2-base architecture; v3/v2.5/v2.6 honor `PerformanceOptions.force_recompute_layer`, which is now configured on the batched finetuning inference engine via `fit_from_preprocessed` and forwarded to each model call with chunkwise inference disabled for backprop.
